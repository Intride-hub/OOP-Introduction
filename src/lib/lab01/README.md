## Разработка класса car

В первую очередь определяем ключевые характеристики машины

- марка(brand)
- скорость(speed)
- вместимость(capacity)
- количество топлива(fuel)

атрибут класса - Transport

Для валидации:
- марка - является строкой и не пуста
- скорость - число и не отрицательно
- вместимость - число и не отрицательно
- количество топлива - число и не отрицательно


Инкапсуляция и контроль доступа
- Закрытые поля в атрибутах(с префиксом _) чтобы избеежать прямого изменения атрибутов
- Доступ через @property
- изменение скорости(speed) через setter с валидацией


Магические методы

* str - строковое представление для пользователя
* repr - техническое представление объекта
* eq - сравнение обьектов (по скорости и марки машины)

Бизнес логика


три простых, базовых методов

1. start_engine() - включает двигатель(если топлива < 0 выбрасывает ошибку)
2. stop_engine() - выключает двигатель
3. drive() - езда (при отключенном двигателе выбрасывает ошибку, расход топлива 10км/1л, при нехватке топлива ошибка)
