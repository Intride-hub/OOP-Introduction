# Лабораторная работа №3 — Наследование и иерархия классов

---

## 1. Цель работы

Освоить механизм **наследования классов** в Python.  
Научиться строить **иерархию объектов** на основе существующего класса из ЛР-1.  
Понять разницу между базовым и производным классом, реализовать **полиморфизм** и **фильтрацию по типу** через коллекцию из ЛР-2.

---

## 2. Описание реализованной иерархии классов

### Иерархия

```
Car  (base.py — базовый класс из ЛР-1/ЛР-2)
 ├── ElectricCar   (электромобиль)
 ├── TruckCar      (грузовик)
 └── SportCar      (спорткар)
```

### Базовый класс `Car` (`base.py`)

Перенесён из ЛР-2 (`model.py`) без дублирования кода.  
Определяет общий интерфейс:

| Метод | Описание |
|---|---|
| `display()` | Строковое отображение объекта (переопределяется в потомках) |
| `calculate()` | Расчёт числового показателя (переопределяется в потомках) |
| `start_engine()` / `stop_engine()` | Управление двигателем |
| `drive(distance)` | Поездка с расходом топлива |

### Дочерние классы (`models.py`)

| Класс | Новые атрибуты | Новые методы | `calculate()` |
|---|---|---|---|
| `ElectricCar` | `battery_capacity` (кВт·ч), `charge_level` (%) | `charge(amount)` | запас хода = `battery_capacity * charge_level / 10` |
| `TruckCar` | `payload` (кг), `trailer_attached` (bool) | `attach_trailer()`, `detach_trailer()` | `payload * speed * (1.2 если прицеп)` |
| `SportCar` | `top_speed` (км/ч), `turbo` (bool) | `activate_turbo()` | `top_speed * (2 если turbo)` |

### Коллекция `Fleet` (`collection.py`)

Расширяет коллекцию из ЛР-2 — теперь хранит объекты **любых** дочерних классов.  
Добавлены методы фильтрации по типу:

- `get_only_electric()` — только `ElectricCar`
- `get_only_trucks()` — только `TruckCar`  
- `get_only_sport()` — только `SportCar`

## 3. Демонстрация работы

### Сценарий 1 — Создание объектов, `display()`

Создаются 4 объекта разных типов. Метод `display()` переопределён в каждом классе:

```
[Car] BMW (id=1): 200 км/ч, 60 л, двигатель заглушен
[ElectricCar] Tesla (id=2): 250 км/ч, батарея 100.0 кВт·ч, заряд 80.0%, двигатель заглушен
[TruckCar] Scania (id=3): 120 км/ч, грузоп. 20000 кг, с прицепом, двигатель заглушен
[SportCar] Ferrari (id=4): 320 км/ч, макс. 400 км/ч, TURBO, двигатель заглушен
```

### Сценарий 2 — Полиморфный `calculate()`

Один и тот же метод вызывается для всех типов — разное поведение:

```
BMW          → calculate() = 1000.0
Tesla        → calculate() = 800.0
Scania       → calculate() = 2880000.0
Ferrari      → calculate() = 800.0
```

### Сценарий 3 — `isinstance()` проверка типов

```
BMW         : Car=True, ElectricCar=False, TruckCar=False, SportCar=False
Tesla       : Car=True, ElectricCar=True,  TruckCar=False, SportCar=False
Scania      : Car=True, ElectricCar=False, TruckCar=True,  SportCar=False
Ferrari     : Car=True, ElectricCar=False, TruckCar=False, SportCar=True
```

### Сценарий 4 — Коллекция `Fleet` и фильтрация по типу

```
Всего в автопарке: 4 машин

-- Только электромобили --
  [ElectricCar] Tesla ...

-- Только грузовики --
  [TruckCar] Scania ...

-- Только спорткары --
  [SportCar] Ferrari ...
```

### Сценарий 5 — Полиморфизм без условий

Единый цикл вызывает `display()` и `calculate()` без каких-либо `if type == ...`:

```python
for vehicle in fleet:
    print(vehicle.display())   # разный вывод для каждого типа
    print(vehicle.calculate()) # разный расчёт для каждого типа
```

### Сценарий 6 — Уникальные методы потомков

```
Tesla: заряжаем батарею +15%
  До зарядки:  заряд 80.0%
  После зарядки: заряд 95.0%

Scania: прицепляем/отцепляем прицеп
  До: [TruckCar] Scania (id=3): 120 км/ч, грузоп. 20000 кг, с прицепом ...
  После отцепки: [TruckCar] Scania (id=3): 120 км/ч, грузоп. 20000 кг, без прицепа ...
```

## 4. Вывод

В ходе лабораторной работы было изучено:

- **Наследование** — как строить иерархию классов на основе существующего класса, используя `super()` без дублирования кода.
- **Переопределение методов** — методы `display()` и `calculate()` определены в базовом классе, но реализованы по-разному в каждом потомке.
- **Полиморфизм** — один и тот же вызов `vehicle.calculate()` даёт разный результат в зависимости от типа объекта, без условных операторов.
- **isinstance()** — позволяет проверять принадлежность объекта к классу с учётом иерархии наследования.
- **Интеграция с коллекцией** — коллекция `Fleet` из ЛР-2 была расширена для работы с разными типами объектов и фильтрации по типу.
